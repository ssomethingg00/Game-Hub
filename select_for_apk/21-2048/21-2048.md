## 1. Snapshot

2048 is a single-player sliding-tile number puzzle played on a 4×4 grid. The player combines matching numbered tiles by sliding them in four directions; equal tiles merge into their sum, and the goal is to produce a tile worth 2048. The game is celebrated for its "one more move" addictiveness, minimal interface, and satisfying merge feedback. It is a landmark example of how a simple mechanic can generate deep strategic play through emergent complexity.

## 2. Core Loops

- **30-second loop:** Press one arrow key (or swipe) → all tiles slide in that direction → matching tiles merge → new tile spawns in a random empty cell → repeat.
- **Session loop:** Start a fresh 4×4 board with two tiles → build toward 2048 tile → optionally continue pushing for 4096, 8192, etc. → game ends when board fills with no merges available. Typical session: 5–20 minutes.
- **Meta loop:** Best score persisted across sessions → player returns to beat their personal record; community leaderboards/sharing provide external motivation. No energy gate or daily limit — fully self-directed re-engagement.

## 3. Mechanics, Controls & Game States

### Core Mechanic
- 4×4 grid of cells; cells are empty or contain a power-of-2 tile (2, 4, 8, 16 … 131072).
- Player chooses a direction (Up/Down/Left/Right).
- **Slide rule:** Every tile slides as far as possible in the chosen direction.
- **Merge rule:** When two identical tiles collide while sliding, they merge into one tile worth their sum. The merged tile **cannot merge again in the same move** (one merge per tile per move). [Confirmed]
- **Critical edge case:** [2, 2, 2, 2] → [4, 4] (leading pair merges first in direction of motion, trailing pair merges second). [Confirmed]
- **Move validity:** A move only counts (tile spawns) if at least one tile moved or merged. [Confirmed]
- After each valid move, one new tile spawns in a random empty cell: value 2 (90% probability) or 4 (10% probability). [Confirmed]
- **Score:** Increases by the value of each merged tile each move. Merging two 32s → +64 to score. [Confirmed]

### Game Modes
| Mode | Description |
|---|---|
| Classic | Standard 4×4 play to 2048 then keep-going |
| Keep Going | Continues after 2048 tile reached; no new win condition |
| (Optional) 5×5 / 6×6 | Larger boards — fan variant, not original |

- Orientation: Portrait-first on mobile; square board adapts to any aspect ratio.

### Win / Lose / Fail States
| State | Trigger | Response |
|---|---|---|
| **Win** | A merged tile reaches value 2048 | Show "You Win!" overlay; offer "Keep Playing" or "New Game" |
| **Keep Going** | Player clicks Keep Playing after win | Remove overlay, suppress future win-trigger, continue play |
| **Game Over** | Board is full AND no adjacent equal tiles exist (horizontally or vertically) | Show "Game Over!" overlay with final score and "Try Again" button |
| **Mid-game** | Normal play | Continue; no penalty for suboptimal moves |

### Game-Over Detection Algorithm
1. Check if any empty cell exists → if yes, game is NOT over.
2. For each cell, check if the cell to its right has the same value → if any match, NOT over.
3. For each cell, check if the cell below has the same value → if any match, NOT over.
4. If none of the above → Game Over. [Confirmed]

### Feedback Systems
- Tile pop/scale animation on spawn (tile "appears" with a small bounce).
- Tile merge animation (brief scale-up pulse on merged tile).
- Color-coded tiles per value (warm palette, darker for higher values).
- Score counter increments visually with each merge.
- Optional: WebAudio merge/spawn sounds, mute toggle.

## 15. Pacing & Difficulty

### Early Game (tiles 2–64)
- Board fills slowly; many empty cells.
- Merges happen frequently; score climbs fast.
- Almost any strategy works.
- **Aha moment:** First 128 or 256 tile — player sees the exponential growth.

### Mid Game (tiles 128–512)
- Board tightens; planning 2–3 moves ahead becomes necessary.
- Corner strategy emerges (keep highest tile in one corner).
- **Key churn point:** Players who don't adopt a strategy stall here; board fills, game over at ~2000–5000 points. [Estimated]

### Late Game (tiles 1024–2048)
- Board is nearly full most of the time; each move is critical.
- One wrong slide can scatter the high tile and cause irreversible board lock.
- **Aha moment:** First 2048 tile — win condition triggered.
- **Keep-going phase:** Post-2048 play; asymptotically harder; rewards expert players.

### Difficulty Curve
- No artificial difficulty scaling; difficulty is entirely self-imposed by board state.
- Randomness (tile spawns) introduces variance — bad luck can end games even with good play.
- Average player reaches 1024–2048 after 5–10 games. Reaching 4096+ requires consistent strategy. [Estimated]

### Common Churn Points
1. First 5 games: unfamiliar with merge priority → quick game-overs.
2. ~500–1000 score range: board fills without reaching 2048 for first time.
3. Post-win: some players stop after 2048; keep-going option retains experienced players.

### Hardest Parts / Risks
1. **Slide + merge correctness:** The "one merge per tile per move" rule with the [2,2,2,2]→[4,4] case is the most bug-prone logic. Test all edge cases exhaustively.
2. **Smooth animation + state sync:** Animating tiles while keeping JS board state in sync (especially merge animation showing the old tile disappearing and new merged tile appearing) requires careful DOM lifecycle management.
3. **Mobile swipe vs scroll conflict:** Touch swipe handler must `preventDefault()` on touchmove to stop page scrolling, but this must not break tap interactions with buttons.
4. **Move validity without visual glitch:** Must detect whether the board actually changed before spawning — a naive re-render on every key press would show false spawns.
5. **Tile identity for animation:** To animate tiles sliding (rather than just re-rendering), each tile needs a persistent ID tracked through moves/merges.

## 17. Open Questions
